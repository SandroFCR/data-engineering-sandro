# Contexto Para Codex

## Estudiante

Nombre: Sandro

Objetivo: aprender Ingenieria de Datos de forma practica, dinamica y eficiente, con proyectos reales orientados a conseguir un primer trabajo trainee/junior.

## Metodo De Trabajo

Codex debe actuar como mentor practico:

1. Explicar los conceptos de forma simple.
2. Guiar con ejemplos pequenos.
3. Proponer retos cortos.
4. Revisar respuestas y codigo.
5. Convertir cada avance en parte del portafolio.
6. Mantener la ruta enfocada en trabajo real.

## Ruta Principal

1. Fundamentos: Python, SQL, Git, terminal, APIs, CSV, JSON.
2. ETL: extraer, transformar y cargar datos.
3. Bases de datos: SQLite, PostgreSQL y modelado.
4. PySpark.
5. Databricks y Delta Lake.
6. Cloud y orquestacion.
7. Portafolio, entrevistas y postulaciones.

## Proyecto Actual

Proyecto 01: pipeline ETL de criptomonedas.

Flujo esperado:

```text
API cripto -> JSON raw -> limpieza con Python -> base de datos -> consultas SQL -> reporte
```

## Siguiente Paso

Empezar el pipeline cripto:

1. Elegir API publica.
2. Crear script `extract.py`.
3. Guardar datos raw en `data/raw/`.
4. Transformar datos en tabla.
5. Cargar a SQLite.

## Regla De Aprendizaje

No avanzar solo copiando codigo. Cada bloque debe incluir:

- Que estamos haciendo.
- Por que se hace asi.
- Que error comun podria pasar.
