# Malla De Aprendizaje - Ingenieria De Datos

## Fase 0 - Exploracion

Objetivo: entender el rol y elegir una direccion clara.

- Que hace un Data Engineer.
- Diferencia entre Data Analyst, Data Engineer, Analytics Engineer y Data Architect.
- Conceptos: ETL, ELT, batch, streaming, data warehouse, data lake, lakehouse.
- Herramientas del mercado: Python, SQL, Git, PostgreSQL, Spark, Databricks, Airflow, cloud.

Entrega: mapa personal de la ruta y glosario inicial.

## Fase 1 - Fundamentos

Objetivo: poder manipular datos y trabajar como desarrollador principiante.

- Python para datos.
- SQL.
- Git y GitHub.
- Terminal.
- APIs.
- Archivos CSV, JSON y Parquet.
- Entornos virtuales.

Entrega: pipeline local con Python y SQL.

## Fase 2 - Data Engineering Core

Objetivo: construir pipelines ETL/ELT confiables.

- Extraccion de datos.
- Limpieza y transformacion.
- Carga a base de datos.
- PostgreSQL.
- Modelado dimensional.
- Data quality.
- Logging y manejo de errores.
- Tests basicos de datos.

Entrega: mini data warehouse con tablas fact y dim.

## Fase 3 - Big Data Y Databricks

Objetivo: trabajar con datos grandes usando una plataforma moderna.

- Spark y PySpark.
- DataFrames.
- Spark SQL.
- Databricks notebooks.
- Delta Lake.
- Arquitectura medallion: bronze, silver y gold.
- Jobs y workflows.
- Optimizacion basica.

Entrega: proyecto lakehouse en Databricks.

## Fase 4 - Cloud Y Orquestacion

Objetivo: automatizar pipelines en un entorno cercano al mundo real.

- AWS, Azure o GCP basico.
- Storage: S3, ADLS o GCS.
- Airflow o Dagster.
- Variables de entorno y secretos.
- Monitoreo.
- Costos basicos.

Entrega: pipeline programado end-to-end.

## Fase 5 - Portafolio Y Trabajo

Objetivo: estar listo para postular.

- GitHub limpio.
- README profesional.
- Diagramas de arquitectura.
- CV tecnico.
- Simulacros de entrevista.
- SQL challenge.
- Preguntas de Python, Spark, Databricks y arquitectura.

Entrega: portafolio con 2 o 3 proyectos fuertes.

## Timeline Realista

- Mes 1 a 3: base fuerte en Python, SQL y ETL.
- Mes 4 a 5: PySpark y Databricks.
- Mes 6: primer portafolio y postulaciones iniciales.
- Mes 7 a 9: proyectos mas fuertes, entrevistas y mejora continua.
