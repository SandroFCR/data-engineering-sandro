# Dia 05 - README De Portafolio

## Que Mejoramos

Hoy mejoramos la presentacion del proyecto para que pueda entenderse mejor en GitHub y en una entrevista.

Un proyecto de portafolio no solo debe tener codigo. Tambien debe explicar:

- que problema resuelve;
- como esta organizado;
- que herramientas usa;
- como se ejecuta;
- que decisiones tecnicas se tomaron;
- que aprendiste.

## Idea Clave

Un README es parte del proyecto.

Si el codigo funciona pero nadie entiende como usarlo o por que existe, el proyecto pierde fuerza en portafolio.

## Secciones Agregadas

- Flujo del pipeline.
- Decisiones tecnicas.
- Como explicarlo en entrevista.
- Preguntas que el proyecto ayuda a responder.
- Proximas mejoras.

## Explicacion Para Entrevista

```text
Construí un pipeline ETL modular para datos de criptomonedas. El pipeline extrae precios desde la API de CoinGecko, guarda la respuesta raw en JSON, transforma los datos a un CSV tabular y los carga en SQLite. Luego ejecuta consultas SQL de análisis y validaciones de calidad para revisar nulos, métricas inválidas y duplicados por snapshot.
```
